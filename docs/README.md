# Docs Layout

This directory is the canonical home for project notes, plans, and reference material.

Subdirectories:

- `plans/`: active implementation plans and working proposals
- `reference/`: durable documentation about current workflow, file roles, and operating procedures
- `deferred/`: postponed proposals or follow-up work that is intentionally not active yet
- `archive/`: completed or superseded notes moved out of active circulation

Current migration intent:

- new planning notes should be created under `docs/plans/`
- durable workflow documentation should live under `docs/reference/`
- older root-level `.md` notes should be reviewed and moved into `docs/` in a separate cleanup pass
